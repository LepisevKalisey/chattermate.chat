# Руководство по развертыванию ChatterMate на Coolify

В данном руководстве подробно описан процесс развертывания платформы ChatterMate на собственном сервере Coolify с использованием подготовленного файла конфигурации Docker Compose.

---

## Архитектура развертывания

Развертывание ChatterMate состоит из следующих сервисов:
1. **frontend**: Веб-интерфейс и скрипт виджета (сервируется через Nginx на порту 80).
2. **backend**: API-сервер на FastAPI (Gunicorn на порту 8000).
3. **knowledge_processor**: Фоновый воркер для обработки базы знаний ИИ.
4. **db**: PostgreSQL 16 с расширением `pgvector` (для векторного поиска).
5. **redis**: Хранилище сессий и брокер сообщений для Socket.IO.

Мы рекомендуем использовать **однодоменную конфигурацию**, где Nginx на фронтенде проксирует запросы на `/api/` и `/socket.io/` напрямую на бэкенд внутри закрытой сети Docker. Это исключает проблемы с CORS и упрощает настройку SSL.

---

## Шаг 1: Создание ресурса в Coolify

1. Откройте панель управления Coolify.
2. Перейдите в ваш **Проект** (Project) и выберите нужную **Среду** (Environment, например, Production).
3. Нажмите кнопку **+ New** (Добавить новый ресурс).
4. Выберите **Docker Compose** в качестве источника развертывания.
5. Укажите имя приложения (например, `chattermate`).
6. Вставьте содержимое файла [docker-compose.coolify.yml](../../docker-compose.coolify.yml) в редактор конфигурации Docker Compose в Coolify.

---

## Шаг 2: Настройка доменов в Coolify

### Вариант А: Однодоменное развертывание (Рекомендуется)
Назначьте публичный домен только для сервиса **frontend**.
1. В настройках ресурса Chattermate в Coolify найдите сервис **frontend**.
2. В поле **Domains** укажите ваш домен, например: `https://chattermate.domain.com`.
3. Для сервиса **backend** домен указывать **не нужно** (он будет доступен извне через прокси-правила Nginx фронтенда).

### Вариант Б: Двухдоменное развертывание (Раздельные поддомены)
Если вы хотите, чтобы бэкенд и виджеты работали на отдельном поддомене:
1. Для сервиса **frontend** укажите домен: `https://chattermate.domain.com`.
2. Для сервиса **backend** укажите домен: `https://api.chattermate.domain.com`.
3. Убедитесь, что вы настроили CORS на бэкенде, добавив адрес фронтенда.

---

## Шаг 3: Настройка переменных окружения

В панели Coolify в разделе **Environment Variables** добавьте следующие переменные:

### Обязательные переменные бэкенда (backend и knowledge_processor)

| Переменная | Значение / Описание | Пример |
|------------|---------------------|--------|
| `DATABASE_URL` | Строка подключения к PostgreSQL | `postgresql+psycopg://postgres:postgres@db:5432/chattermate` |
| `JWT_SECRET_KEY` | Случайная строка для шифрования токенов авторизации | *Сгенерируйте надежный ключ* |
| `CONVERSATION_SECRET_KEY` | Случайная строка для шифрования токенов чата | *Сгенерируйте надежный ключ* |
| `ENCRYPTION_KEY` | Ключ шифрования (Base64 32-байтного ключа Fernet) | *Используйте значение по умолчанию или сгенерируйте новое* |
| `CORS_ORIGINS` | Разрешенные источники запросов (массив JSON) | `["https://chattermate.domain.com"]` |
| `FRONTEND_URL` | Полный URL фронтенда | `https://chattermate.domain.com` |
| `VITE_WIDGET_URL` | Полный URL для загрузки виджета | `https://chattermate.domain.com` (Вариант А) или `https://api.chattermate.domain.com` (Вариант Б) |

### Обязательные переменные фронтенда (frontend)

| Переменная | Значение / Описание | Пример |
|------------|---------------------|--------|
| `VITE_API_URL` | URL API-эндпоинта бэкенда | `https://chattermate.domain.com/api/v1` (Вариант А) или `https://api.chattermate.domain.com/api/v1` (Вариант Б) |
| `VITE_WS_URL` | URL сервера WebSockets (Socket.IO) | `wss://chattermate.domain.com` (Вариант А) или `wss://api.chattermate.domain.com` (Вариант Б) |
| `VITE_WIDGET_URL` | URL скрипта виджета | `https://chattermate.domain.com` (Вариант А) или `https://api.chattermate.domain.com` (Вариант Б) |

> [!TIP]
> В Coolify все добавленные переменные окружения автоматически пробрасываются в сервисы Compose благодаря конструкции `VARIABLE_NAME=${VARIABLE_NAME}` в нашем YAML-файле.

## Шаг 4: Особенности работы с базой данных и pgvector

По умолчанию в [docker-compose.coolify.yml](../../docker-compose.coolify.yml) включен контейнер `db`, использующий [Dockerfile.postgres](../../Dockerfile.postgres).
* При первом запуске он автоматически скачивает репозиторий `pgvector`, собирает его и подключает расширение. Это гарантирует полную работоспособность векторного поиска из коробки.
* Все данные БД сохраняются в именованном томе `postgres_data`, управляемом Coolify.

Если вы захотите использовать внешнюю БД (например, сервис Supabase или RDS):
1. Закомментируйте или удалите сервис `db` в файле Compose.
2. Измените `DATABASE_URL` на строку подключения вашей внешней БД.
3. **Критически важно:** убедитесь, что во внешней базе данных создана и включена поддержка расширения `vector` (`CREATE EXTENSION IF NOT EXISTS vector;`).

---

## Шаг 5: Запуск и проверка развертывания

1. После настройки переменных окружения и доменов нажмите **Deploy** (Развернуть) в панели Coolify.
2. В логах сборки вы увидите:
   * Сборку и установку зависимостей для Vue 3 (фронтенд).
   * Компиляцию Nginx-образа.
   * Сборку бэкенда на Python 3.12.
   * Сборку расширения `pgvector` в PostgreSQL.
3. Бэкенд выполнит скрипт `backend/scripts/start.sh`, который:
   * Автоматически распарсит `DATABASE_URL` для определения хоста.
   * Дождется готовности базы данных.
   * Запустит автоматические миграции БД (`alembic upgrade head`).
   * Начнет обрабатывать запросы через Gunicorn.
4. После успешного завершения деплоя перейдите по домену вашего фронтенда. Вы должны увидеть интерфейс входа в ChatterMate.
