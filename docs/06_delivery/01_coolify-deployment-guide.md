# Руководство по развертыванию ChatterMate на Coolify (chat.lexi.kz)

В данном руководстве подробно описан процесс развертывания платформы ChatterMate на вашем сервере Coolify с использованием подготовленного файла конфигурации Docker Compose, настроенного для сборки из исходников на платформе AMD64.

---

## Архитектура развертывания

Развертывание ChatterMate состоит из следующих сервисов:
1. **frontend**: Веб-интерфейс и скрипт виджета (собирается из исходников и сервируется через Nginx на внутреннем порту 80).
2. **backend**: API-сервер на FastAPI (собирается из исходников, работает через Gunicorn на порту 8000).
3. **knowledge_processor**: Фоновый воркер для обработки базы знаний ИИ.
4. **db**: PostgreSQL 16 с расширением `pgvector` (автоматически собирается из исходников для AMD64).
5. **redis**: Хранилище сессий и брокер сообщений для Socket.IO.

Мы используем **однодоменную конфигурацию** на домене `chat.lexi.kz`. Фронтенд Nginx принимает все запросы и проксирует `/api/` и `/socket.io/` на бэкенд внутри закрытой сети Docker. Это решает проблемы с CORS, устраняет необходимость настраивать отдельные SSL-сертификаты для API и упрощает развертывание.

---

## Шаг 1: Создание ресурса в Coolify

1. Откройте панель управления Coolify.
2. Перейдите в ваш **Проект** (Project) и выберите нужную **Среду** (Environment, например, Production).
3. Нажмите кнопку **+ New** (Добавить новый ресурс).
4. Выберите **Docker Compose** в качестве источника развертывания.
5. Укажите имя приложения (например, `chattermate`).
6. Вставьте содержимое файла [docker-compose.coolify.yml](../../docker-compose.coolify.yml) в редактор конфигурации Docker Compose в Coolify.

---

## Шаг 2: Настройка доменов в Coolify

Поскольку используется однодоменная схема, назначьте домен только для сервиса **frontend**:
1. В настройках ресурса Chattermate в Coolify перейдите к сервису **frontend**.
2. В поле **Domains** укажите ваш домен: `https://chat.lexi.kz`.
3. Для сервиса **backend** и других сервисов публичные домены указывать **не нужно** — они надежно закрыты внутри сети Docker и доступны извне только через Nginx.

---

## Шаг 3: Настройка переменных окружения

В панели Coolify в разделе **Environment Variables** добавьте следующие переменные. Они будут автоматически проброшены в соответствующие контейнеры:

### Переменные бэкенда (backend и knowledge_processor)

| Переменная | Значение / Описание | Пример значения для chat.lexi.kz |
|------------|---------------------|----------------------------------|
| `DATABASE_URL` | Строка подключения к PostgreSQL | `postgresql+psycopg://postgres:postgres@db:5432/chattermate` |
| `JWT_SECRET_KEY` | Ключ шифрования токенов авторизации | *Сгенерируйте случайную длинную строку* |
| `CONVERSATION_SECRET_KEY` | Ключ шифрования токенов чата | *Сгенерируйте случайную длинную строку* |
| `ENCRYPTION_KEY` | Ключ шифрования Fernet (Base64 32-байтного ключа) | *Используйте значение по умолчанию или сгенерируйте новое* |
| `CORS_ORIGINS` | Разрешенные источники запросов (массив JSON) | `["https://chat.lexi.kz"]` |
| `FRONTEND_URL` | Полный URL фронтенда | `https://chat.lexi.kz` |
| `VITE_WIDGET_URL` | Полный URL для загрузки виджета | `https://chat.lexi.kz` |

### Переменные фронтенда (frontend)

| Переменная | Значение / Описание | Значение для chat.lexi.kz |
|------------|---------------------|---------------------------|
| `VITE_API_URL` | URL API-эндпоинта бэкенда | `https://chat.lexi.kz/api/v1` |
| `VITE_WS_URL` | URL сервера WebSockets (Socket.IO) | `wss://chat.lexi.kz` |
| `VITE_WIDGET_URL` | URL скрипта виджета | `https://chat.lexi.kz` |

## Шаг 4: Сборка образов под архитектуру AMD64

Поскольку официальные образы на Docker Hub собраны только для ARM-архитектуры, а ваш сервер работает на AMD64, [docker-compose.coolify.yml](../../docker-compose.coolify.yml) полностью переведен на локальную сборку образов:

* **Бэкенд и Воркер** собираются на основе локального `Dockerfile.backend.prod` под управлением Python 3.12-slim.
* **Фронтенд** собирается на основе `Dockerfile.frontend.prod` (Node 22-slim для сборки Vite-приложения и Nginx в качестве финального веб-сервера).
* **База данных** собирается на основе `Dockerfile.postgres`, который в процессе сборки компилирует расширение `pgvector` непосредственно на вашем AMD64-сервере.

Все образы будут скомпилированы непосредственно в процессе развертывания силами сборщика Coolify. Это гарантирует 100% совместимость с процессором вашего сервера и отсутствие ошибок "exec format error".

---

## Шаг 5: Запуск развертывания

1. После настройки переменных окружения и домена нажмите **Deploy** (Развернуть) в панели Coolify.
2. Сборщик Coolify скачает ваш репозиторий, выполнит сборку всех Dockerfile и запустит контейнеры.
3. Бэкенд выполнит скрипт `backend/scripts/start.sh`, который:
   * Автоматически распарсит `DATABASE_URL` для определения хоста.
   * Дождется готовности PostgreSQL.
   * Запустит автоматические миграции БД (`alembic upgrade head`).
   * Запустит FastAPI бэкенд.
4. После успешного деплоя перейдите на страницу `https://chat.lexi.kz` для работы с системой.
